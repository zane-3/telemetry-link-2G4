## 🔧 一、固件烧录

### 1.1 准备工作

- **烧录工具：** ST-Link V2 或兼容工具
- **烧录软件：** STM32 ST-LINK Utility 或 STM32CubeProgrammer
- **目标芯片：** STM32F103CBT6
- **固件文件：** `build\application\application.hex`

### 1.2 烧录步骤

**所有设备（主站和从站）都烧录相同固件**

1. 连接 ST-Link 到目标板的 SWD 接口（SWDIO、SWCLK、GND、3.3V）
2. 打开烧录软件
3. 选择固件文件：`build\application\application.hex`
4. 点击"Program"或"下载"按钮
5. 等待烧录完成（约 5-10 秒）
6. **观察LED**：烧录成功后上电，系统LED应该立即开始500ms周期闪烁
7. 断开 ST-Link，重复烧录其他设备

> ⚠️ **注意**：
> - 无需为主站和从站烧录不同固件，角色通过串口配置！
> - LED 正常闪烁表示固件运行正常
> - 首次启动时 LED 在 E28 配置期间（约6秒）也会持续闪烁

## ⚙️ 三、设备配置

### 3.1 配置主站

#### 配置命令（依次发送）

```
CFG ROLE=MASTER
CFG ID=1
CFG CH=24
CFG FILTER=1
CFG ACK=1
CFG SAVE
```

#### 命令说明

| 命令 | 说明 | 响应 |
|------|------|------|
| `CFG ROLE=MASTER` | 设置为主站角色 | `OK` |
| `CFG ID=1` | 设置本机ID为1 | `OK` |
| `CFG CH=24` | 设置信道为24（十进制，对应0x18） | `OK` |
| `CFG FILTER=1` | 启用帧过滤（目标ID匹配） | `OK` |
| `CFG ACK=1` | 启用ACK应答机制 | `OK` |
| `CFG SAVE` | 保存配置到Flash | `OK` |

> ⚠️ **注意**：`CFG SAVE` 会保存配置并重启，重启后会按新配置重新配置E28模块，需等待约3-5秒完成。

#### 验证配置

发送查询命令：
```
CFG?
```

**预期返回：**
```
CFG ROLE=MASTER ID=0001 REMOTE=0001 CH=0018 OPTION=0004 FILTER=1 ACK=1 E28=FFFF
```

**关键检查点：**
- ✅ `ROLE=MASTER`
- ✅ `E28=FFFF`（主站使用广播地址）
- ✅ `FILTER=1 ACK=1`

```

```

```
CFG ROLE=SLAVE ID=0001 REMOTE=0001 CH=0018 OPTION=0004 FILTER=1 ACK=1 E28=0001

```

**关键检查点：**
- ✅ `ROLE=SLAVE`
- ✅ `E28=0001/0002/0003`（从站使用唯一地址）
- ✅ `ID` 必须不同（1、2、3...）

## ✅ 五、功能测试

### 5.1 快速测试步骤

**准备：** 1个主站 + 2个从站，所有设备上电

#### 测试1：主站广播

1. 在主站串口发送（十六进制）：
   ```
   A5 04 FF 01 10 20 D3
   ```

2. **预期结果：**
   - ✅ 从站1串口输出：`A5 04 FF 01 10 20 D3`
   - ✅ 从站2串口输出：`A5 04 FF 01 10 20 D3`
   - 📝 两个从站都收到相同数据

#### 测试2：主站单播到从站1

1. 在主站串口发送：
   ```
   A5 04 01 01 11 21 D8
   ```

2. **预期结果：**
   - ✅ 从站1串口输出：`A5 04 01 01 11 21 D8`
   - ❌ 从站2串口无输出（被过滤）
   - ✅ 主站收到ACK：`A5 05 01 01 11 A1 00 xx`

#### 测试3：主站单播到从站2

1. 在主站串口发送：
   ```
   A5 04 02 01 12 22 DB
   ```

2. **预期结果：**
   - ❌ 从站1串口无输出（被过滤）
   - ✅ 从站2串口输出：`A5 04 02 01 12 22 DB`
   - ✅ 主站收到ACK：`A5 05 01 02 12 A2 00 xx`

### 5.2 帧格式说明

```
+------+-----+--------+-----+-----+-----+---------+-----+
| HEAD | LEN | TARGET | SRC | SEQ | CMD | PAYLOAD | CRC |
+------+-----+--------+-----+-----+-----+---------+-----+
  0xA5   1B    1B      1B    1B    1B    0~N       1B
```

**字段说明：**
- **HEAD**: 固定0xA5（帧头）
- **LEN**: 长度（从TARGET到PAYLOAD的字节数）
- **TARGET**: 目标ID（0xFF=广播，0x01=从站1，0x02=从站2...）
- **SRC**: 源ID（发送方ID）
- **SEQ**: 序列号（用于ACK匹配）
- **CMD**: 命令字节（bit7=1表示ACK）
- **PAYLOAD**: 业务数据（可选）
- **CRC**: 校验和（所有前面字节的和，取低8位）

**CRC计算示例：**
```
帧：A5 04 01 01 11 21
CRC = (0xA5 + 0x04 + 0x01 + 0x01 + 0x11 + 0x21) & 0xFF
    = 0xD8
完整帧：A5 04 01 01 11 21 D8
```

### 5.3 手工验收记录

生产验收以串口工具手工发送 HEX 帧为准，建议记录以下信息：

| 项目 | 记录内容 |
|------|----------|
| 主站串口 | COM 号、`CFG?` 返回内容 |
| 从站串口 | COM 号、`CFG?` 返回内容 |
| 广播测试 | 所有从站是否收到 `TARGET=0xFF` 帧 |
| 单播测试 | 只有目标从站收到，非目标从站无输出 |
| ACK测试 | 主站是否收到目标从站 ACK |
| 断电重启 | 重启后 `CFG?` 是否保持原配置 |

如需重复测试，可使用串口工具的 HEX 发送记录或宏功能保存本节中的测试帧。

## 🔄 六、配置修改

### 6.1 修改设备ID

如需修改从站ID（例如从1改为3）：

```
CFG ID=3
CFG SAVE
```

### 6.2 切换主从角色

**从站改为主站：**
```
CFG ROLE=MASTER
CFG SAVE
```

**主站改为从站：**
```
CFG ROLE=SLAVE
CFG ID=2
CFG SAVE
```

### 6.3 修改信道

如需避免干扰，可修改信道：

```
CFG CH=25
CFG SAVE
```

> ⚠️ **注意**：所有设备（主站和从站）必须使用相同信道！

### 6.4 恢复出厂设置

如需重置所有配置：

```
CFG DEFAULT
CFG SAVE
```

**默认配置：**
- 角色：SLAVE（从站）
- ID：1
- 信道：0x18 (24)
- 过滤：启用
- ACK：启用

## 🔧 八、故障排查

### 8.1 配置命令无响应

**可能原因：**
1. 串口未连接到 USART2
2. 波特率不正确
3. 固件未正确烧录

**解决方法：**
1. 检查硬件连接（TX/RX是否交叉）
2. 确认串口参数为 115200-8-N-1
3. 重新烧录固件

### 8.2 从站收不到数据

**可能原因：**
1. 信道不一致
2. E28模块未正确配置
3. 过滤功能错误

**解决方法：**
1. 所有设备发送 `CFG?` 检查信道是否一致
2. 发送 `CFG SAVE` 保存并重启，重新配置E28
3. 检查 `FILTER=1` 且 `TARGET` 匹配

### 8.3 所有从站都收到单播数据

**可能原因：**
1. 从站未启用过滤功能
2. E28地址配置错误

**解决方法：**
1. 发送 `CFG FILTER=1`
2. 发送 `CFG?` 检查 `E28` 地址是否唯一
3. 重新执行 `CFG SAVE`

### 8.4 没有ACK响应

**可能原因：**
1. ACK功能未启用
2. 从站离线或故障
3. 使用了广播帧（广播不要求ACK）

**解决方法：**
1. 发送 `CFG ACK=1`
2. 检查从站是否上电且正常运行
3. 确认发送的是单播帧（TARGET ≠ 0xFF）

### 8.5 配置不保存

**可能原因：**
1. 未执行 `CFG SAVE`
2. Flash写入失败

**解决方法：**
1. 修改配置后必须执行 `CFG SAVE`
2. 断电重启后发送 `CFG?` 验证
3. 如果仍不保存，可能是硬件故障

## 📚 十、附录

### 10.1 完整命令参考

| 命令 | 参数 | 说明 | 示例 |
|------|------|------|------|
| `CFG?` | 无 | 查询当前配置 | `CFG?` |
| `CFG ROLE=` | MASTER/SLAVE | 设置主从角色 | `CFG ROLE=MASTER` |
| `CFG ID=` | 1~254 | 设置本机ID | `CFG ID=2` |
| `CFG REMOTE=` | 1~254 | 设置默认远端ID | `CFG REMOTE=1` |
| `CFG CH=` | 0~99 或 0x00~0xFF | 设置信道 | `CFG CH=24` |
| `CFG OPTION=` | 0~99 或 0x00~0xFF | 设置E28选项 | `CFG OPTION=4` |
| `CFG FILTER=` | 0/1 | 关闭/启用过滤 | `CFG FILTER=1` |
| `CFG ACK=` | 0/1 | 关闭/启用ACK | `CFG ACK=1` |
| `CFG SAVE` | 无 | 保存配置到Flash | `CFG SAVE` |
| `CFG DEFAULT` | 无 | 恢复默认配置 | `CFG DEFAULT` |

### 10.2 配置模板

**快速配置脚本（Windows批处理）**

创建文件 `config_master.bat`：
```batch
@echo off
echo CFG ROLE=MASTER > COM3
timeout /t 1
echo CFG ID=1 > COM3
timeout /t 1
echo CFG CH=24 > COM3
timeout /t 1
echo CFG FILTER=1 > COM3
timeout /t 1
echo CFG ACK=1 > COM3
timeout /t 1
echo CFG SAVE > COM3
timeout /t 1
echo Done!
```
