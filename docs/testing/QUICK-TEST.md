# CRSF 测试模式 - 快速验证指南

## 🎯 目标
验证 2.4G 从站能正常输出 CRSF 信号给飞控，无需主站或无线输入。

## ⚡ 5 分钟快速测试

### 1️⃣ 硬件连接（1分钟）
```
从站 USART2_TX (PA2) → 飞控 UART_RX
从站 GND            → 飞控 GND
从站 5V/3.3V        ← 供电
```

### 2️⃣ 配置飞控（2分钟）

**Betaflight Configurator**:

1. **Ports 页面**
   - 找到连接的 UART (例如 UART3)
   - Serial RX: ✅ 启用
   - 波特率: **420000**
   - 点击 **保存并重启**

2. **Configuration 页面**
   - Receiver Mode: **Serial-based receiver**
   - Serial Receiver Provider: **CRSF**
   - 点击 **保存并重启**

### 3️⃣ 验证结果（2分钟）

**Receiver 页面**:

应该看到：
- ✅ 状态：绿色 "CRSF"
- ✅ CH1: ~1500 (中位)
- ✅ CH2: ~1500 (中位)
- ✅ CH3: ~1000 (**最低，安全**)
- ✅ CH4: ~1500 (中位)
- ✅ CH5: ~1000 (**disarm，安全**)
- ✅ 所有通道稳定不跳动

## ✅ 测试成功标志

1. 接收机类型显示 "CRSF"
2. 绿色连接状态指示
3. 16 个通道全部可见
4. 通道值稳定（误差 < ±5）
5. 油门通道锁定在最低值
6. ARM 通道锁定在关闭状态

## ❌ 常见问题

### 问题：飞控不识别接收机

**解决**:
- 检查接线：TX → **RX** (不是 TX → TX)
- 确认波特率：必须是 **420000**
- 确认 Provider：必须是 **CRSF**
- 重启飞控

### 问题：通道值跳动

**解决**:
- 检查连接线是否松动
- 远离电机和电调（EMI 干扰）
- 尝试降低波特率到 400000

### 问题：油门不是最低

**解决**:
- 这是安全问题！请立即断电
- 检查固件是否正确烧录
- 重新编译并烧录
