# CRSF 测试模式验证报告

## 修改日期
2026-06-10

## 测试模式说明

### 启用的功能
- **CRSF 测试模式**: `APP_CRSF_TEST_MODE_ENABLE = 1`
- **输出周期**: 10ms (100Hz)
- **独立运行**: 无需主站或无线输入

### 测试通道数据
固件会持续输出以下安全的测试值：

| 通道 | 功能 | 值 | 说明 |
|------|------|-----|------|
| CH1 | Roll | 992 | 中位 |
| CH2 | Pitch | 992 | 中位 |
| CH3 | Throttle | **172** | **最低（安全）** |
| CH4 | Yaw | 992 | 中位 |
| CH5 | Arm | **172** | **解锁关闭（安全）** |
| CH6-16 | 其他 | 992 | 中位 |

### CRSF 输出特性
- **波特率**: 420000 bps (USART2)
- **帧率**: ~100 Hz (每 10ms)
- **帧格式**: `0xC8 0x18 0x16 [22 bytes] CRC8`
- **通道范围**: 172-1811 (11-bit CRSF 标准)

## 验证步骤

### 第 1 步：硬件连接
```
从站板 USART2_TX  →  飞控 UART_RX
从站板 GND       →  飞控 GND
从站板电源        ←  5V 或 3.3V (按板卡要求)
```

### 第 2 步：配置 Betaflight

**Ports 页面**:
1. 找到连接的 UART
2. 启用 **Serial RX**
3. 波特率设置为 **420000**
4. 保存

**Receiver 页面**:
1. Receiver Mode = **Serial-based receiver**
2. Serial Receiver Provider = **CRSF**
3. 保存并重启飞控

### 第 3 步：观察结果

**预期显示**:
- ✅ Receiver 类型显示 "CRSF"
- ✅ 连接状态：绿色/已连接
- ✅ CH1 (Roll): ~1500 μs (中位)
- ✅ CH2 (Pitch): ~1500 μs (中位)
- ✅ CH3 (Throttle): ~1000 μs (最低)
- ✅ CH4 (Yaw): ~1500 μs (中位)
- ✅ CH5 (Arm): ~1000 μs (关闭)
- ✅ 通道数据稳定，无抖动

**Betaflight 通道值对应**:
```
CRSF 172  → BF ~1000 μs (最低)
CRSF 992  → BF ~1500 μs (中位)
CRSF 1811 → BF ~2000 μs (最高)
```

## 故障排查

### 问题 1: Betaflight 不识别
**检查**:
- USART2_TX 接到飞控 UART **RX** (不是 TX)
- 波特率确认为 420000
- CRSF provider 已选择
- 从站已正常上电

**调试**:
```bash
# 使用逻辑分析仪查看 USART2 输出
# 应该看到持续的 420000 bps 数据流
```

### 问题 2: 通道值异常
**检查**:
- 所有通道应显示固定值
- CH3 必须为最低
- CH5 必须为 disarm

**原因**:
如果通道值跳动，可能是：
- 波特率误差
- 连接不稳定
- EMI 干扰

### 问题 3: 连接间歇性丢失
**可能原因**:
1. 波特率误差累积
   - 尝试降低到 400000
2. CRC 校验失败
   - 检查 CRSF 实现
3. 电源不稳定
   - 检查供电质量

## 恢复正常模式

如需恢复接收无线 RC 数据的正常模式：

### 方法 1: 修改配置重新编译
编辑 `app_config.h`:
```c
#define APP_CRSF_TEST_MODE_ENABLE      0U  // 改为 0
```

然后重新编译并烧录。

### 方法 2: 使用预编译固件
如果有备份的正常固件，直接烧录即可。
