# Codex 任务：E28-2G4T12S 首次烧录配置流程

## 目标

请基于当前 `zane-3/telemetry-link-2G4` 仓库实现 E28-2G4T12S 数传模块的首次烧录配置流程。

核心要求：

- 第一次烧录 application 后，模块通常仍是出厂配置，配置模式固定使用 `9600 8N1`。
- 仅首次启动时进入 E28 配置模式并写入参数。
- 配置成功后切回透明传输模式，并将 MCU 到 E28 的 UART 改为 `115200 8N1`。
- 之后普通上电/复位不再重复配置，直接进入透明传输模式。

## 手册依据

根据 E28-2G4T12S 产品规格书：

- 配置模式：`M2=1, M1=1, M0=1`。
- 配置模式串口固定：`9600 8N1`。
- 配置指令为 HEX 指令，不是 `AT` / `ATSxxx` 文本命令。
- 传输模式：`M2=1, M1=0, M0=0`。
- 读取参数：连续发送 `C1 C1 C1`。
- 掉电保存参数：发送 `C0 + 5 字节工作参数`，共 6 字节。

## 需要实现的流程

### 1. 首次烧录后第一次启动

1. 检查 MCU Flash 中是否存在本地配置标记。
2. 如果没有标记，说明需要首次配置 E28。
3. 将 E28 模式脚设置为配置模式：

   ```text
   M2=1, M1=1, M0=1
   ```

4. 将 MCU 到 E28 的 UART 临时设置为：

   ```text
   9600 8N1
   ```

5. 等待 `AUX=1`，确认模块空闲。
6. 发送 HEX 配置帧，建议目标配置为：

   ```text
   C0 00 01 28 18 04
   ```

   字节含义：

   | 字节 | 值 | 说明 |
   |---|---|---|
   | HEAD | `C0` | 掉电保存配置 |
   | ADDH | `00` | 地址高字节 |
   | ADDL | `01` | 地址低字节 |
   | SPED | `28` | `8N1 + 115200 串口 + 连传自适应空速` |
   | CHAN | `18` | 信道 `0x18` |
   | OPTION | `04` | 透明传输、推挽输出、12dBm |

7. 再发送读取参数指令：

   ```text
   C1 C1 C1
   ```

8. 判断返回参数中是否包含期望配置。
9. 配置成功后，写入 MCU Flash 本地标记，避免之后重复配置。
10. 切换 E28 到传输模式：

    ```text
    M2=1, M1=0, M0=0
    ```

11. 将 MCU 到 E28 的 UART 改成：

    ```text
    115200 8N1
    ```

12. 进入正常透明传输。

### 2. 之后上电或复位

1. 如果检测到 MCU Flash 本地配置标记，跳过 E28 配置。
2. 直接设置 E28 为传输模式：

   ```text
   M2=1, M1=0, M0=0
   ```

3. 直接使用：

   ```text
   115200 8N1
   ```

4. 不再发送 `C0` 配置帧。

## 必须避免的问题

- 不要使用 `+++`、`AT`、`ATS102=1`、`AT&W` 这类文本命令。
- 不要在 HEX 配置指令后追加 `0D 0A`。
- 不要把模式 0 写成 `M2=0, M1=0, M0=0`。
- 不要每次上电都重新配置 E28。
- 不要在 `AUX=0` 时强行切换或连续发配置命令。

## 建议代码检查点

请重点检查并修改：

- E28 模式脚初始化逻辑。
- MCU 到 E28 的 UART 波特率切换逻辑。
- HEX 配置帧发送逻辑。
- `C1 C1 C1` 读取并确认配置逻辑。
- MCU Flash 配置标记读写逻辑。
- 透传模式下 `USART1 <-> 外部主机/飞控串口` 的转发逻辑。

## 本地验证步骤

### 编译

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1 -Project application
```

### 烧录

```powershell
powershell -ExecutionPolicy Bypass -File .\flash.ps1 -Project application
```

### 首次启动硬件验证

- 观察 M0/M1/M2：应进入配置模式 `1/1/1`。
- 观察 MCU 到 E28 UART：应为 `9600 8N1`。
- 观察发送帧：应发送 `C0 00 01 28 18 04`。
- 观察读取帧：应发送 `C1 C1 C1`。
- 配置成功后，应切换为传输模式 `M2=1,M1=0,M0=0`。
- UART 应改为 `115200 8N1`。

### 第二次上电验证

- 不应再次进入配置模式。
- 不应再次发送 `C0 00 01 28 18 04`。
- 应直接进入传输模式 `M2=1,M1=0,M0=0`。
- UART 应直接为 `115200 8N1`。

### 透传验证

- E28 两端参数一致时，两个模块应能透明传输。
- 外部飞控/主机串口与无线侧双向数据应完整。
- 长时间运行时不应因为 AUX 状态变化误触发重新配置。

## 验收标准

- Application 编译通过。
- 首次烧录后只配置一次 E28。
- 配置成功后本地 Flash 标记有效。
- 后续上电直接进入透明传输。
- 115200 透传稳定。
- 不再存在 AT/ATS 文本配置逻辑。
