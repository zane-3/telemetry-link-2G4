# E28-2G4T12S 配置需求

## 目标

APP 固件需要支持 E28-2G4T12S 2.4G 数传模块的首次启动自动配置和正常运行透传。

核心行为：

- 第一次烧录 application 后，首次启动进入 E28 配置模式。
- 配置模式固定使用 `9600 8N1`。
- 配置命令使用 HEX 帧，不使用 `AT`、`ATSxxx`、`AT&W` 等文本命令。
- 配置成功后写入 MCU Flash 本地标记，后续普通上电或复位不再重复配置。
- 正常透传模式下，MCU 到 E28 的 UART 使用 `115200 8N1`。

## E28 模式脚

| 模式 | M2 | M1 | M0 | 用途 |
|---|---:|---:|---:|---|
| 配置模式 | 1 | 1 | 1 | 读取/写入模块参数 |
| 传输模式 | 1 | 0 | 0 | 透明传输 |

切换模式后需要等待 `AUX=1`，确认模块空闲。

## 配置帧

首次配置写入掉电保存参数：

```text
C0 00 01 28 18 04
```

字节含义：

| 字节 | 值 | 说明 |
|---|---|---|
| HEAD | `C0` | 掉电保存配置 |
| ADDH | `00` | 地址高字节 |
| ADDL | `01` | 地址低字节 |
| SPED | `28` | `8N1 + 115200 串口 + 连传自适应空速` |
| CHAN | `18` | 信道 |
| OPTION | `04` | 透明传输、推挽输出、12dBm |

读取参数命令：

```text
C1 C1 C1
```

读取或写入成功后，返回参数中应包含目标配置：

```text
C0 00 01 28 18 04
```

## 首次启动流程

1. APP 启动后检查 MCU Flash 配置标记。
2. 如果标记不存在，设置 E28 为配置模式 `M2=1,M1=1,M0=1`。
3. 将 USART1 临时切换为 `9600 8N1`。
4. 发送 `C1 C1 C1` 读取参数，确认模块可通信。
5. 发送 `C0 00 01 28 18 04` 写入掉电保存参数。
6. 再次发送 `C1 C1 C1` 读取参数。
7. 只有读取结果匹配目标配置时，才写入 MCU Flash 本地标记。
8. 设置 E28 为传输模式 `M2=1,M1=0,M0=0`。
9. 将 USART1 切换为 `115200 8N1`。
10. 进入正常透明传输。

如果首次配置失败，也必须切回传输模式并把 USART1 设置为 `115200`，避免停留在配置模式。

## 后续上电流程

1. 如果 MCU Flash 配置标记存在，跳过 E28 配置。
2. 设置 E28 为传输模式 `M2=1,M1=0,M0=0`。
3. USART1 保持 `115200 8N1`。
4. 不再发送 `C0` 配置帧。

## 手动配置入口

正常运行时，USART2 收到连续 `+++` 后进入手动配置入口。

要求：

- `+++` 只是 MCU 本地触发序列，不转发给 E28。
- 进入后设置 E28 为配置模式，USART1 使用 `9600 8N1`。
- 固件自动发送 `C1 C1 C1`，并将返回参数以 HEX 文本输出到 USART2。
- 手动配置空闲超时后，自动回到传输模式并恢复 USART1 `115200`。

## 串口透传

正常运行时只启用 USART1 和 USART2 的双向透传：

- USART2 收到的数据转发到 USART1。
- USART1 收到的数据转发到 USART2。
- USART2 收到 `+++` 时不转发到 USART1，而是进入手动配置入口。
- USART3 当前不参与透传链路。

## 验收标准

- Application 主机和从机固件均可编译。
- 首次启动只配置一次 E28。
- 配置成功后本地 Flash 标记有效。
- 后续上电直接进入透明传输。
- 发送的配置帧为 `C0 00 01 28 18 04`，后面不追加 `0D 0A`。
- 读取帧为 `C1 C1 C1`。
- 代码中不再存在 AT/ATS 文本配置逻辑。
- 115200 透明传输稳定。
