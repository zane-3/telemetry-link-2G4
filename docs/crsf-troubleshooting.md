# CRSF 连接故障诊断指南

## 当前问题
- Betaflight 显示警告："接收机协议的 UART 端口必须启用'串行接收机'"
- 通道值不符合预期
- 可能原因：硬件连接或配置不匹配

## 🔍 诊断步骤

### 步骤 1: 确认接线（最重要！）

#### 从站端
```
从站板上找到 USART2 引脚：
- USART2_TX (PA2)  ← 这是输出 CRSF 的引脚
- GND              ← 必须连接
```

#### 飞控端
```
您实际连接到飞控的哪个 UART？

示例：如果连接到飞控 UART3
- 飞控 UART3_RX ← 连接从站 USART2_TX
- 飞控 GND      ← 连接从站 GND
```

#### ⚠️ 常见错误
```
❌ 错误：从站 TX → 飞控 TX
✅ 正确：从站 TX → 飞控 RX

❌ 错误：没有连接 GND
✅ 正确：GND 必须共地
```

---

### 步骤 2: 确认 Betaflight Configuration 页面设置

**接收机选择下拉框**应该是：
- 如果接线到 UART1 → 选择 "UART1"
- 如果接线到 UART2 → 选择 "UART2"
- 如果接线到 UART3 → 选择 "UART3"

**当前截图显示**："串行（通过 UART）" - 这个下拉框的具体选项是什么？

---

### 步骤 3: 确认 Ports 页面配置

**找到实际接线的 UART 行**，例如如果接线到 UART3：

```
UART3 行：
  ✓ 串行接收机：必须勾选
  波特率：115200
```

其他 UART 的"串行接收机"必须**不勾选**。

---

### 步骤 4: LED 指示检查

从站板上应该有 LED 指示灯：
- 正常运行：LED 每 500ms 闪烁一次
- 如果 LED 不闪烁：固件可能没有运行

---

### 步骤 5: 使用万用表/逻辑分析仪

如果有工具，测量从站 USART2_TX (PA2) 引脚：
- 应该有持续的数据输出（电压跳变）
- 每 10ms 一次（100Hz）
- 如果是高电平不变：没有输出
- 如果是低电平不变：可能短路

---

## 🎯 快速测试方法

### 方法 1: 对调测试
如果您有多个 UART 可用：

1. 断开当前连接
2. 改接到飞控的**另一个 UART**
3. 修改 Betaflight Configuration 和 Ports 对应配置
4. 查看是否有效

### 方法 2: 回环测试
验证从站是否真的输出数据：

1. 将从站 USART2_TX 短接到 USART3_RX（如果可用）
2. 使用串口监视器连接到飞控对应的 USB/调试串口
3. 应该能看到数据流

## 🔧 解决方案

### 如果警告依然存在

**最可能的原因**：
1. **Configuration 页面选择的 UART** ≠ **实际接线的 UART**
2. **Ports 页面对应 UART 没有启用串行接收机**

**解决办法**：
1. 物理上确认接线到哪个 UART
2. Configuration 页面下拉框选择该 UART
3. Ports 页面**只在该 UART 启用串行接收机**
4. 保存并重启

## ⚡ 最快验证方法

### 临时测试：使用飞控的 USB 虚拟串口

如果您的飞控支持 USB VCP：

1. 临时将从站 USART2_TX 连接到飞控 **USB VCP 的 RX**
2. Configuration 选择 "USB VCP"
3. Ports 页面 USB VCP 启用串行接收机 @ 115200
4. 测试是否能收到数据

如果这样可以收到，说明：
- ✅ 从站输出正常
- ❌ 之前的 UART 接线或配置有问题
