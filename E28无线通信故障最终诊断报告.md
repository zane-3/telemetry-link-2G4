# E28 一对多无线通信故障诊断报告

**日期：** 2026-06-08
**固件版本：** v1.3
**测试人员：** Kiro AI Assistant

---

## 📋 问题总结

**现象：** 主机和从机之间完全无法进行无线通信，所有测试均显示"无数据"。

---

## ✅ 已确认正常的部分

1. **MCU固件运行正常**
   - 配置模式工作正常（`+++` 进入，`CFG?` 查询）
   - 串口通信正常（115200波特率）
   - LED指示正常（500ms跳转）

2. **配置参数正确**
   - 主机：`ROLE=MASTER ID=0002 CH=0024 FILTER=1 ACK=1 E28=FFFF`
   - 从机：`ROLE=SLAVE ID=0001 CH=0024 FILTER=1 ACK=1 E28=0001`
   - 信道一致：CH=24（BCD格式0x24）

3. **E28配置过程执行**
   - 启动信息显示：`E28 CONFIGURING...`
   - 配置成功：`E28 CONFIG OK`
   - 重配计数器：`E28 RECONFIG COUNT: 0001`

## 🔍 根本原因分析

经过多轮测试和诊断，问题定位在：

### 最可能的原因（90%）

**E28模块的AT配置命令未能真正写入模块内部Flash**

#### 证据：
1. MCU显示 `E28 CONFIG OK`，说明AT命令序列执行完成
2. MCU从E28读取的配置（通过`AT+ADDR?`等）返回正确值
3. 但实际无线通信完全不工作

#### 推测：
- E28模块响应了AT命令（`OK`），MCU认为配置成功
- 但E28内部Flash写入失败，或者配置后需要特殊的保存/重启步骤
- E28实际运行时使用的是旧配置（信道18？），而不是新配置（信道24）

#### 验证方法：
1. **手动配置E28模块**（绕过MCU）
   - 使用USB转TTL直接连接E28模块
   - 发送AT命令：
     ```
     AT
     AT+ADDR=0001
     AT+CH=24
     AT+OPTION=04
     AT+SAVE
     AT+RST
     ```
   - 查询确认：`AT+ADDR?`, `AT+CH?`
   - 如果手动配置后通信正常，则问题在MCU的E28配置代码

2. **检查E28的SAVE命令**
   - E28-2G4M27S模块可能需要特殊的保存步骤
   - 查阅E28数据手册确认正确的配置流程

---

### 次要可能原因（10%）

#### 1. E28模块硬件故障
- 发射电路损坏
- 天线断路或短路
- 供电不足（发射时需要120mA+）

#### 2. 信道理解错误
- BCD编码可能理解错误
- `CFG CH=24` → MCU认为是0x24，但E28理解成其他值

#### 3. 物理环境问题
- 设备距离太远（>100米）
- 金属屏蔽或强干扰
- 天线方向错误

---

## 🔧 推荐的修复步骤

### 步骤1：手动验证E28模块（最优先）

**目标：** 绕过MCU，直接测试E28硬件是否正常

**工具：** USB转TTL模块（波特率9600）

**操作：**
1. 断开MCU与E28的连接
2. USB转TTL连接到E28：
   - TX → E28 RX
   - RX → E28 TX
   - GND → GND
   - VCC → 3.3V

3. 串口终端发送：
   ```
   AT               → 应返回 OK
   AT+VER           → 查看版本
   AT+ADDR?         → 查看当前地址
   AT+CH?           → 查看当前信道
   ```

4. 手动配置从机E28：
   ```
   AT+ADDR=0001
   AT+CH=24
   AT+OPTION=04
   AT+SAVE
   AT+RST
   ```

5. 重复以上步骤配置主机E28：
   ```
   AT+ADDR=FFFF
   AT+CH=24
   AT+OPTION=04
   AT+SAVE
   AT+RST
   ```

6. 测试透传：
   - 主机端输入任意文本
   - 从机端应该收到相同文本

**预期结果：**
- 如果手动配置后通信正常 → 问题在MCU代码
- 如果手动配置后通信仍失败 → E28硬件故障

---

### 步骤2：检查E28配置代码

**如果步骤1证明E28硬件正常，则需要修改MCU代码：**

#### 问题点1：E28的SAVE命令
当前代码可能缺少明确的保存步骤。E28-2G4M27S可能需要：
```c
// 配置完成后
App_E28SendCommand("AT+SAVE\r\n");
HAL_Delay(200);
App_E28SendCommand("AT+RST\r\n");
HAL_Delay(1000);
```

#### 问题点2：验证逻辑
`App_E28ResponseMatchesConfig()` 可能只检查E28的响应，但没有真正验证无线通信是否工作。

#### 建议的改进：
1. 在配置E28后，强制发送 `AT+SAVE` 和 `AT+RST`
2. 延长复位后的等待时间（至少1秒）
3. 添加无线回环测试（主从机互发测试帧）

---

### 步骤3：检查硬件连接

**检查项目：**
1. E28模块的供电电压（应为3.0-3.6V）
2. E28的TX/RX是否连接到正确的MCU引脚
3. E28的M0/M1引脚状态（应为00 = 透传模式）
4. 天线是否正确连接（50Ω阻抗匹配）
5. 两个设备的距离（先测试<1米）

---

### 步骤4：固件代码修改

如果需要修改代码，重点检查以下函数：

#### `App_E28WriteConfig()`
检查是否完整发送了所有AT命令，包括SAVE和RST。

#### `App_QueryRadioConfig()`
确认能正确读取E28配置。

#### `App_E28ResponseMatchesConfig()`
验证逻辑是否严格。

```

AT+CH=XX          - 设置信道（00-99 BCD）

AT+OPTION=XX      - 设置选项

AT+ADDR?          - 查询地址
AT+CH?            - 查询信道

```
